配置指南

约 1098 字大约 4 分钟

RWKV Agent Kit 提供了灵活的配置系统，支持多种配置方式和丰富的配置选项。本指南将详细介绍如何配置和优化您的智能体。

📋 配置文件格式

RWKV Agent Kit 支持 TOML、JSON 和 YAML 格式的配置文件：

TOML 格式 (推荐)

# config.toml
[model]
path = "./models/rwkv-4-world-7b-v1-20230626-ctx4096.pth"
device = "cuda"  # 或 "cpu"
precision = "fp16"  # 或 "fp32"
max_tokens = 2048
temperature = 0.7
top_p = 0.9
top_k = 40

[memory]
type = "sqlite"
path = "./data/agent_memory.db"
max_context_length = 4096
memory_threshold = 0.8
embedding_model = "sentence-transformers/all-MiniLM-L6-v2"

[database]
url = "sqlite://./data/agent.db"
max_connections = 10
connection_timeout = 30

[logging]
level = "info"
file = "./logs/agent.log"
max_file_size = "10MB"
max_files = 5

[tools]
enable_builtin = true
custom_tools_dir = "./tools"

[performance]
thread_pool_size = 4
batch_size = 8
cache_size = 1000

JSON 格式

{
  "model": {

🎛️ 配置选项详解

模型配置 (model)

选项类型默认值描述
pathString必需RWKV 模型文件路径
deviceString"cpu"运行设备："cpu", "cuda", "mps"
precisionString"fp32"精度："fp16", "fp32"
max_tokensu322048最大生成令牌数
temperaturef320.7温度参数 (0.0-2.0)
top_pf320.9Top-p 采样参数
top_ku3240Top-k 采样参数
repetition_penaltyf321.1重复惩罚

记忆配置 (memory)

选项类型默认值描述
typeString"sqlite"存储类型："sqlite", "postgresql"
pathString"./memory.db"SQLite 数据库路径
urlString-PostgreSQL 连接 URL
max_context_lengthu322048最大上下文长度
memory_thresholdf320.7记忆存储阈值
embedding_modelString"all-MiniLM-L6-v2"嵌入模型名称
max_memoriesu3210000最大记忆数量

数据库配置 (database)

选项类型默认值描述
urlString必需数据库连接 URL
max_connectionsu3210最大连接数
connection_timeoutu6430连接超时（秒）
idle_timeoutu64600空闲超时（秒）
enable_loggingboolfalse启用 SQL 日志

日志配置 (logging)

选项类型默认值描述
levelString"info"日志级别："trace", "debug", "info", "warn", "error"
fileString-日志文件路径
max_file_sizeString"10MB"最大文件大小
max_filesu325最大文件数量
formatString"json"日志格式："json", "text"

🚀 性能优化配置

GPU 加速配置

[model]
device = "cuda"
precision = "fp16"  # 使用半精度提高速度
batch_size = 8      # 批处理大小

[performance]
gpu_memory_fraction = 0.8  # GPU 内存使用比例
enable_tensor_cores = true # 启用 Tensor Cores

内存优化配置

[memory]
max_context_length = 1024  # 减少上下文长度
memory_threshold = 0.9     # 提高记忆阈值
max_memories = 5000        # 限制记忆数量

[performance]
cache_size = 500           # 减少缓存大小
thread_pool_size = 2       # 减少线程数

高并发配置

[database]
max_connections = 50
connection_timeout = 10

[performance]
thread_pool_size = 16
batch_size = 16
max_concurrent_requests = 100

🔧 环境变量配置

您可以使用环境变量覆盖配置文件中的设置：

# 模型配置
export RWKV_MODEL_PATH="/path/to/model.pth"
export RWKV_DEVICE="cuda"
export RWKV_PRECISION="fp16"

# 数据库配置
export RWKV_DATABASE_URL="sqlite://./data/agent.db"

# 日志配置
export RWKV_LOG_LEVEL="debug"
export RWKV_LOG_FILE="./logs/debug.log"

在代码中使用：

use rwkv_agent_kit::config::AgentConfig;

// 自动从环境变量加载配置
let config = AgentConfig::from_env()?;

// 或者合并配置文件和环境变量
let config = AgentConfig::from_file("config.toml")?
    .merge_env()?;

📊 配置验证

验证配置文件

use rwkv_agent_kit::config::{AgentConfig, ConfigError};

match AgentConfig::from_file("config.toml") {
    Ok(config) => {
        // 验证配置
        if let Err(e) = config.validate() {
            eprintln!("配置验证失败: {}", e);
            return;
        }
        println!("配置验证成功");
    }
    Err(ConfigError::FileNotFound) => {
        eprintln!("配置文件不存在");
    }
    Err(ConfigError::ParseError(e)) => {
        eprintln!("配置文件解析错误: {}", e);
    }
    Err(e) => {
        eprintln!("其他配置错误: {}", e);
    }
}

🔍 故障排除

常见配置问题

  1. 模型加载失败

    错误: 无法加载模型文件
解决: 检查模型路径是否正确，文件是否存在

  2. GPU 内存不足

    错误: CUDA out of memory
解决: 减少 batch_size 或使用 fp16 精度

  3. 数据库连接失败

    错误: 无法连接到数据库
解决: 检查数据库 URL 和权限设置

性能调优建议

  1. CPU 优化

    • 设置合适的 thread_pool_size
    • 使用 fp32 精度
    • 减少 batch_size

  2. GPU 优化

    • 使用 fp16 精度
    • 增加 batch_size
    • 启用 tensor_cores

  3. 内存优化

    • 减少 max_context_length
    • 限制 max_memories
    • 调整 cache_size
